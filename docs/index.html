Triple Inverted Pendulum

Step 1 — Define the system

1

Describe the physical setup

We describe the whole system with just four numbers called generalised coordinates. Instead of tracking the X and Y position of every single point, we only need:

  • x — where the cart is on the rail (metres, positive = right)
  • θ₁ — how far link 1 is tilted from straight-up (radians)
  • θ₂ — tilt of link 2
  • θ₃ — tilt of link 3

Everything else (the XY position of every joint and mass) can be calculated from these four numbers.

## ── STEP 1: Define the system parameters ───────────────────────────────

# All the physical constants you can change to test different setups
M_cart = 2.0   # kg — how heavy is the cart?
l1, m1 = 0.30, 0.15   # link 1: length (m), mass (kg)
l2, m2 = 0.25, 0.10   # link 2
l3, m3 = 0.20, 0.08   # link 3
g = 9.81               # gravity (m/s²)

# The state of the whole system is 8 numbers:
#   positions: x, theta1, theta2, theta3
#   velocities: x_dot, theta1_dot, theta2_dot, theta3_dot
state = [x, theta1, theta2, theta3,
         x_dot, theta1_dot, theta2_dot, theta3_dot]

Step 2 — Physics: equations of motion

2

Calculate accelerations from forces (Lagrangian mechanics)

The Lagrangian method gives us one master equation that automatically produces all four equations of motion. It works from energy instead of drawing free-body diagrams:

T = kinetic energy (motion) V = potential energy (height) L = T − V

After doing the calculus (see README for full derivation), the result is a compact matrix equation:
M(q) · q̈ = F_ext − Coriolis(q, q̇) − Gravity(q) − Damping

  • M(q) — 4×4 mass matrix (how inertia is spread across all joints)
  • Coriolis — centrifugal / coupling forces when links spin fast
  • Gravity — torque gravity applies to each tilted link
  • F_ext — cart force from the motor (our control handle)

We solve for the accelerations by rearranging:

## ── STEP 2: Compute accelerations at every timestep ────────────────────

def equations_of_motion(state, F_cart):
    x, th1, th2, th3, x_dot, th1_dot, th2_dot, th3_dot = state

    # ── 2a. Build the 4×4 mass matrix M ──────────────────────────────────
    # (how hard it is to accelerate each coordinate)
    # Diagonal = self-inertia; off-diagonal = coupling between links
    M = build_mass_matrix(th1, th2, th3)   # depends on current angles

    # ── 2b. Coriolis / centrifugal vector h ───────────────────────────────
    # These appear because the mass matrix changes as the links rotate.
    # Think: a spinning link "flings" the one above it sideways.
    # They are proportional to the SQUARE of angular velocities (θ_dot²),
    # so they are small near the upright and large during fast swings.
    h = coriolis_vector(th1, th2, th3, th1_dot, th2_dot, th3_dot)

    # ── 2c. Gravity torque vector G ───────────────────────────────────────
    # Gravity pulls each leaning link further away from vertical.
    # G[i] = -alpha_i * g * sin(theta_i)
    # When theta > 0 (leaning right), G < 0 → pulls it further right.
    # This is the INSTABILITY — gravity is a destabilising force here.
    G = gravity_vector(th1, th2, th3)

    # ── 2d. Damping (friction) ────────────────────────────────────────────
    D = [cart_friction * x_dot,
         joint_damping * th1_dot,
         joint_damping * th2_dot,
         joint_damping * th3_dot]

    # ── 2e. Solve  M · q_ddot = F_ext - h - G - D  ───────────────────────
    F_ext = [F_cart, 0, 0, 0]   # motor force acts only on the cart
    rhs = F_ext - h - G - D
    q_ddot = solve(M, rhs)      # numpy.linalg.solve — fast, no explicit M⁻¹

    # Returns [x_ddot, th1_ddot, th2_ddot, th3_ddot,
    #          x_dot,  th1_dot,  th2_dot,  th3_dot ]
    return q_ddot + [x_dot, th1_dot, th2_dot, th3_dot]

Step 3 — Motor model

3

Convert motor command → cart force

The GoBilda Yellow Jacket brushed DC motor follows a simple linear torque-speed curve. The faster the cart already moves, the less force the motor can deliver (back-EMF effect). A lead screw then converts shaft rotation into linear cart force.

  • Primary motor: 1120 RPM → max 0.149 m/s (8 mm lead)
  • Secondary motor: 312 RPM → max 0.042 m/s, but 2× the torque
## ── STEP 3: Motor — shaft torque → cart force ───────────────────────────

FREE_SPEED_RPM  = 1120          # no-load speed at full voltage
STALL_TORQUE_Nm = 2.8           # maximum torque (at zero speed)
LEAD_m          = 0.008         # 8 mm lead screw (metres per revolution)
EFFICIENCY      = 0.85          # thread friction loss

def motor_force(command, cart_velocity):
    """
    command : float in [-1, +1]  (PWM duty cycle from controller)
    cart_velocity : current cart speed in m/s

    Returns: force applied to cart in Newtons
    """
    # Convert cart speed back to shaft speed
    omega_free   = FREE_SPEED_RPM * 2*pi / 60   # rad/s
    omega_shaft  = cart_velocity / (LEAD_m / (2*pi))  # rad/s

    # Torque-speed curve:  tau = tau_stall * (command - omega_shaft/omega_free)
    # At stall (omega_shaft=0): tau = tau_stall * command  (max torque)
    # At free-run (omega_shaft=omega_free): tau = 0         (no torque)
    tau = STALL_TORQUE_Nm * (command - omega_shaft / omega_free)

    # Lead screw: F = tau * (2*pi / lead) * efficiency
    force = tau * (2*pi / LEAD_m) * EFFICIENCY
    return force

Step 4 — Controller (LQR)

4

Compute the control force (LQR — optimal automatic gains)

A plain PD controller is very hard to tune for a triple pendulum — the three links couple together in complex ways. Instead we use LQR (Linear Quadratic Regulator), which automatically calculates the best possible gains by solving a single equation.

LQR linearises the physics at the upright position (where sin θ ≈ θ), then finds the gain matrix K that minimises a cost function balancing how much the links lean vs. how much force we use. 

## ── STEP 4: LQR controller ──────────────────────────────────────────────

# ── 4a. Linearise the physics at the upright equilibrium ─────────────────
# At theta=0: sin(theta)≈theta, cos(theta)≈1, Coriolis≈0
# The result is a simple linear system:   z_dot = A*z + B*u
A, B = linearise_at_upright(params)   # 8×8 and 8×1 matrices

# ── 4b. Choose cost weights ───────────────────────────────────────────────
# Q = how much do we care about each state being non-zero?
# R = how much do we care about using large forces?
#       State:   [ x,   th1,   th2,   th3,  x_dot, th1_dot, th2_dot, th3_dot ]
Q = diagonal(    [10,  2000,  2000,  2000,     1,     100,     100,     100  ])
R = 0.001   # small = controller is allowed to use large forces

# ── 4c. Solve the Riccati equation for the gain matrix K ─────────────────
# scipy finds the matrix P such that:
#   A^T P + P A - P B R^-1 B^T P + Q = 0
# Then: K = R^-1 * B^T * P   (shape: 1 × 8)
P = scipy.linalg.solve_continuous_are(A, B, Q, R)
K = (1/R) * B.T @ P

# ── 4d. At every timestep, apply the optimal control law ─────────────────
def lqr_force(state, x_ref=0.0):
    z = state.copy()
    z[0] -= x_ref          # express cart position relative to target
    force = -(K @ z)       # single matrix-vector multiply → one number
    return clamp(force, -MAX_FORCE, +MAX_FORCE)

# All 8 gains are found automatically.
# The signs and magnitudes are globally optimal for the chosen Q and R.
# No manual trial-and-error needed.

Step 5 — Run the simulation

5

Step the simulation forward in time

We use a Zero-Order-Hold (ZOH) loop: at every control step the controller picks a force, we hold it constant, and then let the physics integrator advance time with that fixed force. This is exactly what real microcontrollers do (sample → compute → output → wait → repeat). 

## ── STEP 5: Main simulation loop ────────────────────────────────────────

# Starting conditions — pendulum nearly upright, tiny lean on each link
state = [0.0,           # x:  cart at centre
         radians(3),    # th1: 3° lean
         radians(2),    # th2: 2° lean
         radians(1.5),  # th3: 1.5° lean
         0, 0, 0, 0]    # all velocities zero

dt      = 0.01   # control step = 10 ms  (100 Hz)
t_end   = 5.0    # simulate 5 seconds
history = []     # store results for plotting

t = 0.0
while t < t_end:

    # ── 5a. Controller decides the force ─────────────────────────────────
    F_cart = lqr_force(state, x_ref=0.0)   # wants cart at x=0

    # ── 5b. Convert to motor command, then apply back-EMF model ──────────
    command = F_cart / MAX_FORCE             # normalise to [-1, +1]
    F_actual = motor_force(command, state[4])  # real force (back-EMF limited)

    # ── 5c. Integrate physics for this control step ───────────────────────
    # scipy RK45 adaptive solver — automatically picks sub-steps for accuracy
    def ode(t, s):
        return equations_of_motion(s, F_actual)   # F_actual is constant here

    solution = scipy.integrate.solve_ivp(ode, [t, t+dt], state,
                                         method='RK45', rtol=1e-6)
    state = solution.y[:, -1]   # state at end of this step

    # ── 5d. Record results ────────────────────────────────────────────────
    history.append({
        't':     t,
        'x':     state[0],
        'th1':   degrees(state[1]),
        'th2':   degrees(state[2]),
        'th3':   degrees(state[3]),
        'force': F_actual,
    })

    t += dt   # advance clock

# history now contains the full time series — plot or animate it

6

Plot and animate the result

# ── 6b. Animation ─────────────────────────────────────────────────────────
def draw_frame(i):
    cart_x = x_values[i]

    # Work out joint positions from angles (trigonometry)
    pivot1 = [cart_x, 0]
    tip1   = [cart_x + l1*sin(th1[i]),  l1*cos(th1[i])]
    tip2   = [tip1[0] + l2*sin(th2[i]), tip1[1] + l2*cos(th2[i])]
    tip3   = [tip2[0] + l3*sin(th3[i]), tip2[1] + l3*cos(th3[i])]

    # Draw the cart as a rectangle and the three links as lines
    draw_cart(cart_x)
    draw_link(pivot1, tip1, color="blue")
    draw_link(tip1,   tip2, color="green")
    draw_link(tip2,   tip3, color="red")

anim = animation.FuncAnimation(fig, draw_frame,
                                frames=len(t_values),
                                interval=20)   # 20 ms = 50 fps
plt.show()

Putting it all together

→

The complete flow at a glance

## ── Complete pseudocode summary ─────────────────────────────────────────

# 1. Define physical constants (masses, lengths, motor specs)
params = setup_system()

# 2. Pre-compute LQR gain matrix K once (offline, before simulation starts)
A, B  = linearise_physics(params)
K     = solve_riccati(A, B, Q, R)

# 3. Set starting state (nearly upright, small lean)
state = initial_state()

# 4. Simulate in a loop (each iteration = one 10 ms control step)
while t < t_end:

    F = -(K @ state)                     # LQR: one matrix multiply
    F = clamp(F, -MAX_FORCE, MAX_FORCE)  # don't exceed motor limit

    state = integrate_physics(state, F, dt=0.01)   # RK45 ODE solver
    record(t, state, F)
    t += 0.01

# 5. Plot and animate the recorded history
plot_time_series(history)
animate_pendulum(history)
